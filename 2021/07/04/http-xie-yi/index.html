熟悉HTTP协议结构和通讯原理

HTTP协议特点

1.支持客户/服务器模式

客户/服务器模式工作的方式是由客户端向服务器发出请求，服务器端相应请求，并进行相应服务

2.简单快速

客户想服务器请求服务时，只需传送请求方法和路径

.请求方法常用的有GET HEAD POST 每种方法规定了客户与服务器联系的类型不同

由于HTTP协议简单，是的HTTP服务器的程序规模小，因而通信速度很快

3.灵活

HTTP允许传输任意类型的数据对象

正在传输的类型由Content-Type加以标记（Content-Type是HTTP八种用来标识内容类型的标识）

4无连接

限制每次连接只处理一个请求

服务器处理完客户的请求，并收到客户的应答后，即断开连接

采用这种方式可以节省传输时间

kkep-alive 保持存活

5.无状态

HTTP协议是无状态协议

无状态是指协议对于事务处理没有记忆公能力，缺少状态意味着如果后续处理需要前面的信息，则必须重传，这样可能导致每次连接传送的数据量增大

在服务器不需要先前信息时他的应答就比较快

URI与URL

URI：一个紧凑的饿字符串用来标识抽象或物理资源

A URI 可以及你不被分为定位符 名字或者两者都是

属于“Uniform Resource Locator”（URL）是URI的子集，除了确定一个资源，还提供一种定位该资源的主要访问机制（如其网络“位置”）

URI下面由URL URN

URN作用好像一个人的名字，URL像是一个人的地址

URN确定了东西的身份, 然后通过URL提供了找到他的方式

需要注意的是 URL是URI的一种 但并不是所有的URI都是URL

                     URI和URL最大的差别是“访问机制”  URN是唯一标识的一部分 ， 是身份细腻些

​ urn 没有自己的访问方式

HTTP报文结构分析-请求报文

HTTP通用报文头

ACCEPT：作用 浏览器可以接受的媒体类型

ACCept text/html 代表浏览器可以接受服务器回发的类型为txt/html 也就是我们常说的html文档，如果服务器无法返回text/html类型的数据，服务器应该返回以恶406错误（Non Acceptable）

Accept：/ 代表浏览器可以处理所有类型

Connection

HOST 请求报头域主要用于指定被请求资源的Internet主机和端口号，它通常从HTTP URL中提取出来的

User-Agent

告诉HTTP服务器，客户端使用的操作系统和浏览器的名称和版本

很多情况下我们回通过User-Agent来判断浏览器类型，从而进行不同的兼容设计

Cookie与Session

Cookie是在客户端 Session是在服务器端

Cookie实际上是一小段文本信息 客户端请求服务器 如果服务器需要记录该用户状态 就向客户端浏览器办法一个Cookie

客户端浏览器会把Cookie保存起来 当浏览器再请求该网站时 浏览器把请求的网址联通Cookie一同提交给服务器，服务器检查该Cookie 以此来辨认用户状态

在网页栏填写 Javascript:alert（document.cookie）

Cookie工作原理

1.先发起请求 2.服务器set-cookie 3.返回服务器响应结果 4. 客户端进行读取 set-cookie 5.再次发起请求

6.服务器检查cookie 返回响应结果

Session工作原理

Session是了另一种记录客户状态的机制，保存在服务器上，客户端浏览器访问服务器的时候，服务器把客户端信息以某种形式记录在服务器上

保存Session ID的方式

cookie URL重写 隐藏表单（添加一个隐藏字段）

Cookie的有有效期比较长 但是Session的有效期短，容易超时失效

Session主动失效 程序调用HttpSession.inv
